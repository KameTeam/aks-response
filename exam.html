  • Quelles sont les différences entre le langage orienté objet et le style procédural ? Donnez les avantages et les inconvénients de chacun d'entre d'eux.

    Le procécdural nous oriente vers un développement plus lourd, on pourrais comparer la façon d'afficher des tableaux. En procédural on aurais: $tableau = ['id': '1','name': 'geoffrey','phone': '01.01.01.01.01','birthday':'07/11/94']; $tableau2 = ['id': '2','name': 'louise','phone': '01.01.01.01.01','birthday':'07/11/94']; $alltableau = [$tableau, $tableau2]; foreach($alltableau as $onetableau){ echo $onetableau['id'].' - '.$onetableau['name'].' - '.$onetableau['phone'].' - '.$onetableau['birthday']; } En Développement objet nous devons nous orienter vers un développement plus structurer qui nous demande de crée des class, en comparaison avec l'exemple du haut nos tableau se transforme en class User avec plusieurs propriètes, un constructeur, les setteur et getter et les méthodes une foix cette class crée le code du dessus peut s'écrire ainsi $user = new User('geoffrey', '01.01.01.01', '07/11/94') ; $user2 = new User('louise', '01.01.01.01', '07/11/94') ; $allUser = [user, user2]; foreach($allUser as $oneUser){ echo $oneUser->getId().' - '.$oneUser->getName().' - '.$oneUser->getPhone().' - '.$oneUser->getBirthday() } Passons au avantage et inconvénients grâce à un développement objet on peut réutiliser son code dans plusieurs projet, les variable son plus claire et dynamique. On peut ce connecter à une base de donnée. Néanmoin cet façon de développer demande plus d'organisation de fichier mais le code se retrouve ranger et ordonnée ce qui permet une meilleur visibilité du projet lors de mise a jour ou encore d'arriver sur un projet existant, toute ces bénéfice fond du procédural une façon de développer basic des langages ( il est bien pour comprendre/apprendre mais ne permet pas d'utiliser les pleinne capacité du langage )
